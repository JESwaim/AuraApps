# Issue: Improve settings validation and model status UI

**Description**

The settings screen allows users to configure things like the remote API endpoint and manage on-device models. This task is to improve the validation of user input (e.g., for URLs) and provide clearer feedback on the status of models (e.g., not downloaded, downloading, ready).

**Acceptance Criteria**

- [ ] Input fields in the settings screen have proper validation (e.g., checking for a valid URL format).
- [ ] The UI clearly displays the status of each on-device model.
- [ ] Users can easily initiate or cancel model downloads from the settings screen.
- [ ] The app provides clear feedback messages for actions taken in the settings screen (e.g., "Settings saved," "Model download started").

v0.4: Safety + UX
